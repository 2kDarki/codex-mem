# Installation Guide

## Quick Start

Install codex-mem from npm:

```bash
npm install -g @2kdarki/codex-mem
codex-mem codex init
codex-mem codex watch
```

That's it! The Codex-first runtime will automatically:
- Create your default configuration under `~/.Codex-mem/`
- Watch Codex transcript JSONL files for new sessions
- Update `~/.codex/AGENTS.md` with fresh memory context
- Keep the worker service available for memory extraction and search

Start a new Codex session after initialization and memory will begin accumulating automatically.

> **Compatibility note:** Codex is the primary installation path. Claude Code and Cursor remain supported compatibility hosts on the same worker, database, and search pipeline.

## System Requirements

- **Node.js**: 18.0.0 or higher
- **Codex**: Current Codex desktop or CLI sessions writing transcripts under `~/.codex/sessions/`
- **Bun**: JavaScript runtime and process manager (auto-installed if missing)
- **SQLite 3**: For persistent storage (bundled)

## Advanced Installation

For development or testing, you can clone and build from source:

### Clone and Build

```bash
# Clone the repository
git clone https://github.com/thedotmack/claude-mem.git
cd claude-mem

# Install dependencies
npm install

# Build hooks and worker service
npm run build

# Start the worker manually if needed
codex-mem worker start

# Verify worker is running
codex-mem worker status
```

### Post-Installation Verification

#### 1. Automatic Dependency Installation

Dependencies are installed automatically during plugin installation. The SessionStart hook also ensures dependencies are up-to-date on each session start (this is fast and idempotent). Works cross-platform on Windows, macOS, and Linux.

#### 2. Verify Plugin Installation

Check that the transcript watcher configuration exists:
```bash
cat ~/.Codex-mem/transcript-watch-state.json
```

#### 3. Data Directory Location

Data is stored in `~/.Codex-mem/`:
- Database: `~/.Codex-mem/claude-mem.db`
- PID file: `~/.Codex-mem/.worker.pid`
- Port file: `~/.Codex-mem/.worker.port`
- Logs: `~/.Codex-mem/logs/worker-YYYY-MM-DD.log`
- Settings: `~/.Codex-mem/settings.json`

Override with environment variable:
```bash
export CLAUDE_MEM_DATA_DIR=/custom/path
```

#### 4. Check Worker Logs

```bash
npm run worker:logs
```

#### 5. Test Context Retrieval

```bash
npm run test:context
```

## Upgrading

Upgrades are automatic when updating via the plugin marketplace. Key changes in recent versions:

**v7.1.0**: PM2 replaced with native Bun process management. Migration is automatic on first hook trigger.

**v7.0.0+**: 11 configuration settings, dual-tag privacy system.

**v5.4.0+**: Skill-based search replaces MCP tools, saving ~2,250 tokens per session.

See [CHANGELOG](https://github.com/thedotmack/claude-mem/blob/main/CHANGELOG.md) for complete version history.
